# AGENTS.md

Personal scheduling / task-prioritization app. Multi-user, offline-first PWA.
A "dumb" Rust data server syncs rows; all domain logic lives client-side in TypeScript.

## Stack

- Backend: Rust (edition 2021), `axum` 0.8, `sqlx` 0.9 (SQLite). No domain logic:
  auth, row sync (snapshot + per-row LWW), and static serving only. Serves the API
  under `/api` and the built frontend at `/` with SPA fallback.
- Frontend: Preact 10 + TSX, Vite, `@preact/signals`, `preact-iso` routing,
  `vite-plugin-pwa`. Offline-first: IndexedDB cache + optimistic ops + background sync.
- Domain logic: pure TypeScript in `frontend/src/lib` (layout, resolve, reorder,
  insert, resize, run, dependency graph, fractional indexing). Framework-free.
- Types: defined once in Rust `backend/src/types`, exported to TS via `ts-rs` into
  `frontend/src/bindings`. Auth: signed cookies (`axum-extra`), Argon2 hashes.

## Layout

```
backend/                   Rust crate; bins `schedule` (server) and `adduser`.
backend/src/types/         sqlx-free wire/row types; derive serde + ts_rs::TS.
backend/src/models/        sqlx `*Row` structs mapping SQL columns <-> types.
backend/src/sync.rs        apply_op (authorize + LWW + soft-delete) and snapshot/delta.
backend/src/rev.rs         per-user revision counter (the LWW clock).
backend/src/auth.rs        signed-cookie sessions + Argon2 password hashing.
backend/src/db.rs          SqlitePool setup; runs embedded migrations on boot.
backend/src/error.rs       shared error type mapped to HTTP responses.
backend/src/routes/        auth + sync; mounted in main.rs.
backend/migrations/        SQLx migrations, embedded at build time (sqlx::migrate!).
frontend/src/lib/          pure domain logic; framework-free (fractional, fuzzy, timefmt, result).
frontend/src/lib/schedule/ layout, resolve, reorder, insert, resize, run.
frontend/src/lib/project/  task list ordering + dependency graph.
frontend/src/lib/ops/      Operation apply/invert (per-row upsert/delete).
frontend/src/data/         IndexedDB, api client, sync engine.
frontend/src/state/        Preact signals (base/pending/effective, history); mutations/ build ops.
frontend/src/ui/           feature folders + components; CSS Modules.
frontend/src/bindings/     generated by ts-rs (cargo test); not hand-edited, gitignored.
Dockerfile                 Multi-stage: frontend -> musl backend -> Alpine runtime.
```

## Commands

```bash
cargo build --release          # build backend
cargo test                     # Rust tests; also regenerates frontend/src/bindings
cargo fmt && cargo clippy      # format + lint before committing
cd frontend && npm install     # install frontend deps
cd frontend && npm run dev     # Vite dev server :5173 (proxies /api -> :3000)
cd frontend && npm run build   # emit frontend/dist (incl. PWA service worker)
cd frontend && npm test        # Vitest
cd frontend && npm run lint    # ESLint
cd frontend && npm run typecheck
```

Dev: run the backend (`cargo run --bin schedule`) and `npm run dev` in two
terminals. Vite proxies `/api` to `:3000` so cookies behave as in production.
Required server env: `DATABASE_URL`, `APP_SECRET` (64+ hex chars; ephemeral if
unset). See `README.md` for the full env-var table.

## Conventions

- Keep the codebase lean. Prefer deleting over adding. When a change makes code
  obsolete (dead functions, replaced wrappers, stale branches), remove it.
- Comments are sparse: only annotate genuinely tricky logic, not obvious code.
- Comments are concise: 20 words max. No change-explaining or narration comments.
- Don't leave commented-out code or TODOs behind a change.
- TypeScript: branded ids/`OrderKey`, discriminated unions, a small `Result<T,E>`.
  Import area barrels as namespaces (`import * as layout from "@lib/schedule"`).
  `lib/` stays pure (no preact/data/state/ui); enforced by ESLint.

## Gotchas

- The server trusts client-computed values; it enforces only ownership and per-row
  LWW (`updated_rev`). Sync is the only mutation path; undo/redo is client-side.
- Ids are client-minted UUIDv7. Deletes are soft (tombstones via `deleted_rev`).
- `frontend/src/bindings` is generated from `backend/src/types` by `cargo test`
  (ts-rs). Don't hand-edit; field drift surfaces as a TS compile error.
- Migrations are embedded at compile time; rebuild after editing
  `backend/migrations/`.
